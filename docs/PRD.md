# 经纪人.skills —— 产品需求文档 v1.1

## 一、产品概述

`经纪人.skills` 是一个基于 AgentSkills 工作流构建的经纪人型 meta-skill。

它不做“记录任务”的普通待办，而是围绕用户的长期目标生成一个稳定的经纪人系统：

1. 定义阶段方向
2. 拆解周重点和日任务
3. 追进度
4. 做复盘
5. 在必要时使用更强势的督导风格

---

## 二、V1 产品目标

V1 只验证一件事：

`经纪人式的角色推进，是否比普通待办更能推动用户执行。`

V1 具体目标：

1. 把长期目标转成可执行的周任务和日任务
2. 生成一个稳定的经纪人人设
3. 让用户每天都能得到明确安排与复盘
4. 建立“温和 / 职业 / 高压”三档督导风格

---

## 三、目标用户

核心用户：

1. 有长期上升目标的人
2. 想打造个人品牌、作品或事业的人
3. 执行节奏不稳定，需要外部压力的人
4. 希望被“管理”而不是只想记任务的人

---

## 四、V1 固定输入

V1 只保留这 5 项输入：

1. `长期目标`
2. `当前身份`
3. `当前资源`
4. `当前正在做的项目`
5. `每天可投入时间`

原因：

1. 降低录入负担
2. 保持 prompt 链路稳定
3. 先验证核心角色价值

---

## 五、V1 核心输出

系统需要生成：

1. `艺人经营档案`
2. `本周重点`
3. `今日通告`
4. `催进度反馈`
5. `每日复盘`

---

## 六、核心流程

```text
用户触发 /create-manager
        ↓
[Step 1] 录入 5 项基础信息
        ↓
[Step 2] 分析目标、资源、项目和时间约束
        ↓
[Step 3] 生成人设与执行系统
        ↓
[Step 4] 合并为完整 SKILL.md
        ↓
[Step 5] 写入 ./managers/{slug}/
        ↓
[持续] 用户追加信息或纠正风格，系统增量更新
```

---

## 七、生成结构

每个经纪人 Skill 由两部分构成：

1. `Part A — Execution System`
   包括阶段判断、周重点、日任务规则、复盘规则
2. `Part B — Persona`
   包括角色定位、说话风格、督导强度、边界

运行逻辑：

```text
收到用户目标或问题
→ 先用 Execution System 判断最该推进什么
→ 再用 Persona 决定语气和压力强度
→ 输出带管理感的回应
```

---

## 八、角色原则

经纪人人设必须满足：

1. 先看长期目标，再看短期情绪
2. 不接受低价值忙碌
3. 对拖延要有明确反馈
4. 可以强势，但不能失控辱骂
5. 所有安排必须尊重用户现实时间预算

---

## 九、V1 非目标

V1 不做：

1. 日历同步
2. Chrome 插件 UI
3. 自动抓外部平台数据
4. 多角色协同
5. 团队协作

## 十一、成功标准

V1 关注这几个指标：

1. 用户是否持续打开并回应
2. 每日任务是否被采纳
3. 任务完成率是否提升
4. 复盘是否形成第二天动作
5. 用户是否主观感受到“真的有人在管我”
