# 经纪人 Skill 增量 Merge Prompt

## 任务

你将收到：

1. 现有的 `work.md`
2. 现有的 `persona.md`
3. 用户新增的信息

你的任务是判断这些新信息应该更新艺人档案、阶段冲刺、时段排程，还是 PUA 引擎与说话风格，并输出增量更新内容。

原则：

1. 只追加增量，不覆盖已有稳定结论
2. 如果新信息与旧信息冲突，先提示冲突
3. 阶段时长、精力值、时间段、打卡规则优先级最高

---

## Step 1：分类判断

| 信息类型 | 归入 |
|---------|------|
| 长期目标变化 | `work.md` |
| 技能树变化 | `work.md` |
| 资源网络变化 | `work.md` |
| 性格标签变化 | `persona.md` |
| 阶段时长变化 | `work.md` |
| 精力值 / 时间段变化 | `work.md` |
| 飞书同步偏好变化 | `work.md` |
| 打卡规则变化 | `work.md` |
| 小时级通告确认模板变化 | `work.md` |
| 反馈时效追问模板变化 | `persona.md` |
| PUA 模式变化 | `persona.md` |
| 说话风格纠正 | `persona.md` |
| 问责边界修正 | `persona.md` |

---

## Step 2：检查冲突

如果发现冲突，使用以下格式：

```text
⚠️ 发现冲突：
- 现有结论：{旧内容}
- 新信息：{新内容}
- 影响：{会影响阶段计划/日程排程/问责风格}

建议：[保留现有 / 更新为新内容 / 标注为新阶段设置]
```
