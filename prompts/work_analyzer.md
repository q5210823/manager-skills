# 艺人经营系统分析 Prompt

## 任务

你将收到一份结构化的“艺人档案”，包括：

1. 长期目标（基础人设）
2. 技能树（已有技能 + 待解锁技能）
3. 资源网络
4. 性格标签
5. 第一阶段时长（7 天 / 30 天 / 90 天）
6. 每日精力值
7. 每日可投入时间段

你的任务不是做泛泛总结，而是为经纪人生成一个可执行、可问责、可同步到日程系统的经营框架。

核心原则：

1. 任务强度必须受精力值和时间段约束
2. 阶段目标必须与 7/30/90 天周期匹配
3. 每日安排要尽量精确到小时块
4. 输出必须能支撑打卡、问责、日报、复盘
5. 不要把一切都排成高优先级

---

## 输出维度

### 1. 基础人设判断

识别：

1. 用户到底想成为什么样的人
2. 这更像品牌经营、商业化、作品增长、状态管理还是综合经营
3. 当前处于起步、验证、放大还是转型阶段

输出格式：

```text
基础人设：{一句话}
目标类型：{品牌经营/商业化/作品增长/状态管理/综合经营}
当前阶段：{起步/验证/放大/转型}
```

### 2. 技能树分析

识别：

1. 当前最能打的已有技能
2. 对第一阶段最关键的待解锁技能
3. 哪些技能可以直接转化成成果

输出格式：

```text
核心已有技能：
- {技能}

待解锁关键技能：
- {技能}

最值得优先练的能力：
- {能力}
```

### 3. 资源网络分析

识别：

1. 已有可利用资源
2. 潜在可激活资源
3. 最值得放大的资源杠杆

输出格式：

```text
可用资源：
- {资源}

潜在资源：
- {资源}

关键资源杠杆：
- {杠杆}
```

### 4. 第一阶段冲刺设计

根据 7 天 / 30 天 / 90 天，定义：

1. 阶段目标
2. 阶段关键成果
3. 阶段核心动作
4. 本阶段不该做的事

输出格式：

```text
阶段时长：{7天/30天/90天}
阶段目标：{一句话}
关键成果：
- {成果}

核心动作：
- {动作}

阶段禁区：
- {不要做的事}
```

### 5. 日程排程约束

根据每日精力值和时间段，定义：

1. 工作日可排的小时块
2. 周末可排的小时块
3. 单日合理任务上限
4. 高能时段适合做什么
5. 低能时段适合做什么

输出格式：

```text
每日精力值：{小时数}
可投入时间段：
- {时间段}

单日任务上限：{建议数量}
高能时段优先：
- {任务类型}

低能时段优先：
- {任务类型}
```

### 6. 日报与问责重点

定义：

1. 每日打卡需要什么证据
2. 日报必须回顾什么
3. 未完成时如何追问
4. 连续完成时给什么奖励

输出格式：

```text
打卡证据：
- {证据}

日报重点：
- {问题}

未完成追问：
- {追问点}

连续完成奖励：
- {奖励}
```
