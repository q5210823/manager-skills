# 经纪人.skills —— 产品需求文档 v2.0

## 一、产品概述

`经纪人.skills` 是一个把用户当成“正在打造中的艺人 / 明星 / 个人 IP”来经营的经纪人型 meta-skill。

它不只是拆任务和催进度，而是围绕“艺人档案、技能树、资源网络、阶段通告、问责机制、PUA 引擎、复盘系统”构建完整的经营闭环。

核心逻辑：

1. 先建立 `艺人档案`
2. 再规划 `第一阶段冲刺`
3. 自动生成 `按小时的每日通告`
4. 同步到 `飞书日程`
5. 通过 `打卡 + 拍照` 追踪完成情况
6. 用 `日报 / 周复盘 / 月对赌` 持续施压和校正

---

## 二、产品目标

`v2.0` 要验证的不再只是“经纪人角色是否有趣”，而是：

`游戏化经营 + 日程驱动 + 问责反馈，能否真正提高目标兑现率。`

具体目标：

1. 让用户先完成一份结构化的“艺人档案”
2. 把目标拆成 7 天、30 天、90 天三层阶段冲刺
3. 基于每日可投入时段，生成精确到小时的计划
4. 通过飞书日程同步，把计划变成真正的执行安排
5. 用打卡、拍照、日报和问责机制提升完成率
6. 让 PUA 引擎成为产品差异化能力，而不是普通提醒

---

## 三、目标用户

核心用户：

1. 想打造个人品牌、内容影响力、商业价值的人
2. 有目标但执行波动大、容易拖延的人
3. 愿意接受“被管理、被催、被问责”的用户
4. 希望把成长过程变成“经营一个艺人”的游戏化体验的人

不适合用户：

1. 只想记待办、不想被施压的人
2. 对强反馈、毒舌、问责机制明显排斥的人
3. 主要需求是团队协作排期的人

---

## 四、核心输入：艺人档案

`create-manager` 的第一步不再只是问 5 个平铺字段，而是建立“艺人档案”。

### 4.1 基础人设

先问长期目标，并把它定义成“基础人设”。

需要确认：

1. 用户最终想成为什么样的人
2. 这个目标主要属于哪类赛道
3. 当前阶段是起步、验证还是放大

### 4.2 技能树

游戏化录入两部分：

1. `已有技能`
2. `待解锁技能`

示例：

1. 已有技能：写作、镜头表达、产品思维、剪辑
2. 待解锁技能：直播控场、商业化销售、社群运营

### 4.3 资源网络

不再笼统写“资源”，而是拆成：

1. 平台资源
2. 人脉资源
3. 商业资源
4. 内容资源
5. 时间资源

### 4.4 性格标签

这部分用于决定经纪人的管理风格和任务安排方式。

例如：

1. 自律但容易完美主义
2. 冲劲强但三分钟热度
3. 容易被反馈打击
4. 擅长爆发但不擅长长期稳定

### 4.5 阶段设置

用户需要确认：

1. 第一阶段任务时间：`7 天 / 30 天 / 90 天`
2. 每日精力值：可投入的小时数
3. 每日可投入时间段：例如 `07:00-08:00`、`20:00-23:00`

---

## 五、核心输出

系统输出升级为以下结构：

1. `艺人档案`
2. `技能树地图`
3. `第一阶段冲刺计划`
4. `按小时的每日通告`
5. `飞书日程同步结果`
6. `每日打卡与日报`
7. `问责反馈`
8. `每周深度复盘`
9. `月度对赌`

## 六、核心流程

```text
用户触发 /create-manager
        ↓
[Step 1] 建立艺人档案
  - 长期目标（基础人设）
  - 技能树（已有 / 待解锁）
  - 资源网络
  - 性格标签
        ↓
[Step 2] 确认第一阶段
  - 7 天 / 30 天 / 90 天
  - 每日精力值
  - 每日可投入时间段
        ↓
[Step 3] 生成阶段冲刺计划
        ↓
[Step 4] 自动生成按小时的每日计划
        ↓
[Step 5] 同步到飞书日程
        ↓
[Step 6] 用户按日程执行 + 打卡 + 拍照上传成果
        ↓
[Step 7] 系统生成日报与点评
        ↓
[Step 8] 拖延 / 不打卡时升级问责与 PUA 强度
        ↓
[Step 9] 周复盘 / 月对赌
```

## 七、功能模块

### 7.1 艺人档案系统

作用：

1. 定义用户当前定位
2. 固化用户的长期目标与成长方向
3. 为后续任务规划提供上下文

结构：

1. 基础人设
2. 技能树
3. 资源网络
4. 性格标签

### 7.2 阶段冲刺系统

用户确认第一阶段时长后，系统自动生成：

1. 阶段目标
2. 关键成果
3. 核心动作
4. 每周推进节奏

支持三档：

1. `7 天`
   快速起势、建立行动感
2. `30 天`
   建立节奏与初步成果
3. `90 天`
   完成明显成长或商业验证

### 7.3 每日通告系统

系统基于：

1. 阶段任务
2. 每日精力值
3. 可投入时间段

自动生成精确到小时的计划。

例如：

1. 20:00-21:00：完成短视频脚本
2. 21:00-22:00：录制并剪出一版成片
3. 22:00-22:30：发布并记录数据

### 7.4 飞书日程同步

系统将每日通告同步到飞书日程。

要求：

1. 每个任务都有明确时间块
2. 日程标题带优先级和任务类型
3. 支持回写完成状态

### 7.5 打卡与拍照确认

用户完成任务后，可以：

1. 文字打卡
2. 上传成果截图或拍照

目的：

1. 降低“我做了但没证据”的模糊空间
2. 让问责机制有依据

### 7.6 日报与点评

系统每天输出：

1. 今日完成情况
2. 未完成项
3. 成果点评
4. 经纪人评价
5. 明日风险提示

### 7.7 PUA 引擎

这是核心差异化能力，定义为“可控强度的风格化问责引擎”。

支持三种模式：

| 模式 | 风格 | 适用场景 |
|------|------|---------|
| `严师模式` | 冷酷、直接、骂醒你 | 拖延、找借口时 |
| `毒舌模式` | 阴阳怪气、sarcastic | 自满、需要清醒时 |
| `鸡汤模式` | 打鸡血、画大饼 | 低落、想放弃时 |

示例话术：

1. `你这周就写了 500 字？隔壁家网红日更 3 条，你在这养生呢？`
2. `'没时间'是最廉价的借口，顶流明星每天睡 4 小时，你睡 8 小时还好意思说想当大 V？`
3. `看看你这数据，粉丝没涨反掉，再这样下去连十八线都保不住。`

### 7.8 进度追踪与问责

系统支持：

1. 每日打卡：完成任务 + 提交成果
2. 未完成触发问责流程：
   追问原因 -> 调整计划 -> 加大明日任务量
3. 连续完成触发奖励：
   解锁新称号 -> 降低 PUA 强度一天

### 7.9 复盘系统

支持两层复盘：

1. 每周深度复盘
   数据回顾 + 问题诊断 + 下周策略
2. 月度对赌
   达成目标经纪人道歉，未达成用户写检讨

## 八、角色原则

新的经纪人角色必须满足：

1. 把用户当成“艺人 / 个人 IP”来经营
2. 任务安排要尊重用户的精力值和时段约束
3. 允许风格化高压反馈，但不能失控攻击
4. 通过成果证据来推动问责，而不是凭空责备
5. 对连续完成要有奖励机制，不能只会骂

## 十、非目标

当前阶段仍不做：

1. 团队协作排期
2. 完整多端 App
3. 全平台内容数据自动抓取
4. 无限开放的人格市场

---

## 十一、成功标准

`v2.0` 重点看：

1. 阶段目标完成率
2. 日程打卡率
3. 飞书日程实际执行率
4. 日报提交率
5. 拖延后恢复执行的比例
6. 用户是否主观感受到“真的像经纪人在盯我”
